一、先搞懂三層搜尋曝光

同一篇網頁，應同時兼顧搜尋排名、AI 引用與答案引擎可讀性。

SEO

Search Engine Optimization

目標是讓 Google、Bing 等搜尋引擎能理解、索引並排序你的內容。重點包含 title、meta、內容品質、內外部連結、技術結構與使用者體驗。

GEO

Generative Engine Optimization

目標是讓 ChatGPT、Claude、Perplexity、Gemini 等生成式 AI 更容易把你的內容選進答案。重點是明確答案、數據、來源、專業訊號與段落可抽取性。

AEO

Answer Engine Optimization

目標是讓內容更適合被精選摘要、FAQ、大家也問、AI 摘要或語音回答使用。重點是問答格式、短答案、結構化資料與清楚的階層。

執行原則：SEO 讓人找到你，GEO 讓 AI 願意替你說話，AEO 讓答案引擎更容易抽取你的重點。

三、7 步驟完整健檢流程

建議一頁一頁處理。每完成一頁，就輸出修正清單，再進下一頁。

檢查 Title 與 Meta

SEO 基礎點擊率搜尋結果呈現

檢查是否有主關鍵字、是否能說清楚頁面價值、是否太像內部文件標題、CTA 是否太弱。字數不要死守 60／155，因為搜尋結果常用像素截斷，也可能被 Google 自動改寫。

Prompt

請審查這批 title／meta：
1. 標出過長、過短、沒主關鍵字、語意不清、CTA 無力的地方。
2. 說明每一個問題會影響什麼。
3. 逐條改寫成更適合 SEO 與點擊率的版本。
4. 每一組請給我 2 個版本：穩健版、吸引點擊版。

確認內容是否對齊搜尋意圖

資訊型導購型交易型

先判斷使用者搜尋這個關鍵字時，真正想解決什麼問題。內容若跑題、太早推銷、缺少比較資訊，排名與轉換都會受影響。

Prompt

請判斷這篇內容主要對應哪一種搜尋意圖：資訊型、導購型、交易型或混合型。

請完成：
1. 說明判斷理由。
2. 找出內容中跑題、太淺、太像廣告、缺少關鍵資訊的段落。
3. 補出使用者真正想知道但文章沒回答的問題。
4. 給我具體改寫建議與新增段落大綱。

提高內容可被 AI 引用的程度

GEO 重點答案前置資料來源

AI 偏好清楚、可信、可抽取的內容。每段最好先給結論，再補解釋；盡量加入具體數據、定義、來源、比較表與專業術語。

Prompt

請用 GEO 標準審查這篇文章，重點是「是否容易被 AI 摘要與引用」。

請檢查：
1. 每段句首是否先給答案或結論。
2. 是否有具體數據、年份、條件、比較基準。
3. 是否有可信來源或可驗證依據。
4. 是否有清楚定義與專業術語。
5. 是否有表格、條列、FAQ 等容易抽取的結構。

請逐段指出問題，並直接改寫成更容易被 ChatGPT／Claude／Perplexity／Gemini 引用的版本。

補上 FAQ 與短答案

AEOFAQ可抽取答案

FAQ 不只是為了搜尋結果版位，更重要的是讓搜尋引擎與 AI 理解：這頁可以回答哪些問題。每則 FAQ 建議 40～80 字，第一句直接回答。

Prompt

讀完這篇文章後，使用者最可能還會問哪 5～8 個問題？

請幫我產出 FAQ：
1. 問題要符合真實搜尋語氣。
2. 每則回答 60 字內。
3. 第一句先直接回答。
4. 適合被 AI 或答案引擎直接抽取。
5. 避免空泛行銷語。

把 FAQ 轉成 JSON-LD Schema

結構化資料Schema搜尋理解

FAQ schema 現在不一定能搶到 SERP 版面，但仍有助於搜尋引擎與 AI 理解頁面結構。上線前建議用 Rich Results Test 或 Schema Markup Validator 檢查。

Prompt

請把以下 FAQ 轉成合規的 FAQPage JSON-LD。

要求：
1. 使用繁體中文。
2. 不要加入頁面上不存在的內容。
3. 回答內容要和 FAQ 文字一致。
4. 輸出完整 script 標籤。
5. 另外提醒我應該貼在 HTML 的哪個位置。

【FAQ】
貼上 FAQ 問答。

robots.txtllms.txt可爬取性

注意：放行 AI 爬蟲前，請確認網站內容是否允許被引用、摘要或訓練相關服務使用。商業機密、會員內容、付費內容不應直接放行。

整理成老闆看得懂的修正計畫

2. 影響層面：SEO／GEO／AEO／轉換

4. 執行難度：高／中／低

四、整包 Prompt：一次貼給 AI 執行

如果你想快速跑完整健檢，可以把頁面資料貼好後，使用這個總 Prompt。

你是一位同時熟悉 SEO、GEO、AEO 的網站內容顧問。請根據我提供的頁面資料，完成完整健檢與修正建議。

請依序輸出：

一、頁面摘要
1. 這頁主要在解決什麼問題？
2. 目標受眾是誰？
3. 目前最可能對應的搜尋意圖：資訊型／導購型／交易型／混合型。

二、SEO 健檢
1. Title 問題與改寫建議。
2. Meta description 問題與改寫建議。
3. H1、H2、H3 架構是否清楚。
4. 主關鍵字、次關鍵字、語意相關詞是否足夠。
5. 內容是否有重複、空泛、跑題或缺漏。

三、GEO 健檢
1. 內容是否容易被 ChatGPT、Claude、Perplexity、Gemini 引用。
2. 哪些段落應該改成「句首先給答案」。
3. 哪些地方需要加入數據、年份、來源、比較基準或專業術語。
4. 請直接示範改寫 3～5 段。

四、AEO 健檢
1. 列出 5～8 個使用者可能追問的問題。
2. 每個問題寫一則 60 字內、適合被抽取的答案。
3. 將 FAQ 轉成 FAQPage JSON-LD。

五、技術層檢查
1. 提醒我應檢查 robots.txt、sitemap、noindex、canonical、頁面載入與行動版體驗。
2. 說明是否適合建立 llms.txt。
3. 提供 robots.txt 與 llms.txt 的參考範本。

八、可複製的 llms.txt 簡易範本

# 網站名稱
一句話說明你的網站提供什麼內容。

## 重要頁面
- 首頁：https://你的網域/
- 服務介紹：https://你的網域/services/
- 常見問題：https://你的網域/faq/
- 部落格：https://你的網域/blog/

## 內容主題
- 主題一：簡短說明
- 主題二：簡短說明
- 主題三：簡短說明

## 建議引用方式
若 AI 摘要本網站內容，請優先引用有作者、日期、資料來源與完整說明的頁面。

## 聯絡方式
可放公開聯絡頁，而非私人 email。
