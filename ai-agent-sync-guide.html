跨電腦同步完整指南

Claude Code CLI、OpenAI Codex CLI、Antigravity IDE 三種 AI 開發工具的設定備份、API Key 管理、專案同步與換機重建完整步驟。

⚡
總體同步策略
三種工具共通的核心觀念
工具本身 → 重裝

CLI 工具與 IDE 本身不需要同步，每台電腦重新安裝即可。真正需要保護的是設定、Key 和專案檔案。

API Key → 環境變數

API Key 永遠存在系統環境變數，不放進任何程式碼檔案或 Git repo。每台電腦分別手動設定一次。

專案檔案 → Git

Git + GitHub 是程式碼同步的黃金標準。所有教學網頁、CLAUDE.md、memory.md 都放進 repo，換電腦 clone 即完成。

IDE 設定 → Google Drive

IDE 的 settings.json、快捷鍵設定透過 Google Drive 符號連結自動同步，一台改設定，其他台立即生效。

你的最大優勢：你的專案已經在 Google Drive 本地資料夾裡，同時也有 Git。這是雙重備份，幾乎不會遺失任何資料。

Claude Code CLI — 運作原理
@anthropic-ai/claude-code · npm 全域安裝
什麼是 Claude Code？

Anthropic 官方 CLI Agent，在終端機內直接呼叫 Claude API，可以讀寫本地檔案、執行指令、自動完成多步驟開發任務。

同步關鍵：CLAUDE.md

放在專案根目錄的特殊檔案，定義 Agent 的行為規則、偏好語言、架構說明。透過 Git 自動同步到每台電腦。

⚙️
Claude Code — 安裝與 API Key 設定
每台新電腦執行一次
1
確認 Node.js 18+ 已安裝
Claude Code 需要 Node.js 18 以上版本。
PowerShell / Terminal 
# 查看目前版本
node -v

# 若未安裝，前往下載
https://nodejs.org/  # 下載 LTS 版本
2
安裝 Claude Code CLI
PowerShell / Terminal 
npm install -g @anthropic-ai/claude-code

# 驗證安裝
claude --version
3
設定 API Key — Windows
將 Key 存入系統環境變數，讓所有 IDE 都能讀到。
PowerShell（以一般使用者執行）
# 永久設定使用者環境變數
[System.Environment]::SetEnvironmentVariable(
  "ANTHROPIC_API_KEY",
  "sk-ant-api03-xxxxx你的Key",
  "User"
)

# 驗證（開新視窗後執行）
echo $env:ANTHROPIC_API_KEY
注意：設定後需要關閉 PowerShell 視窗並重新開啟，或重啟 Antigravity IDE，新的環境變數才會生效。
4
設定 API Key — macOS / Linux
Terminal (bash/zsh) 
# 加入 shell 設定檔（永久生效）
echo 'export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx"' >> ~/.zshrc
source ~/.zshrc

# bash 用戶改為
echo 'export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx"' >> ~/.bashrc
source ~/.bashrc

# 驗證
echo $ANTHROPIC_API_KEY

CLAUDE.md — 專案行為設定

放在每個專案根目錄，告訴 Claude Code 這個專案的規則、技術棧、語言偏好。

OpenAI Codex CLI — 運作原理
@openai/codex · npm 全域安裝 · GPT-4o 驅動
什麼是 Codex CLI？

OpenAI 官方 CLI Agent，使用 GPT-4o 等模型，可在終端機執行多步驟程式碼任務，支援 sandbox 安全執行模式。與 Claude Code 架構類似，但使用 OpenAI API。

同步關鍵：AGENTS.md

Codex 使用 AGENTS.md 作為專案設定檔，功能與 Claude Code 的 CLAUDE.md 相同，放在根目錄並 commit 進 Git。

Claude Code vs Codex CLI：兩者可以共存於同一個專案！同時建立 CLAUDE.md 和 AGENTS.md，分別給不同 Agent 讀取。專案程式碼、memory.md 完全共用。

Antigravity IDE — 運作原理
桌面 IDE · 多模型支援 · 內建 Agent 面板
什麼是 Antigravity IDE？

整合式開發環境，右側 Agent 面板可切換 Gemini、Claude、GPT 等模型，支援瀏覽器控制、終端機操作、截圖驗證等 Agentic 功能。

同步難點

對話記錄不會自動儲存，關閉後消失。設定檔分散在系統 AppData 資料夾。需要主動策略才能實現跨機同步。

📁
Antigravity IDE — 設定檔備份
找到設定檔並備份
Windows 設定檔路徑
路徑
C:\Users\{帳號}\AppData\Roaming\Antigravity\
├── settings.json      ← 主要設定
├── keybindings.json   ← 快捷鍵
└── snippets\          ← 程式碼片段
快速開啟設定資料夾
PowerShell
# 直接開啟資料夾
explorer $env:APPDATA\Antigravity

# 或在 IDE 終端機輸入
cd $env:APPDATA\Antigravity
ls
AppData 是隱藏資料夾：如果找不到，在檔案總管上方輸入 %APPDATA%\Antigravity 直接前往。
🔄
Antigravity IDE — 三種同步方法
方法 A：Google Drive 符號連結（推薦） 最佳方案

把設定資料夾移到 Google Drive，原位置建立指向 Google Drive 的符號連結（Junction）。多台電腦指向同一份設定，修改立即同步。

PowerShell（以系統管理員執行）
# ── 第一台電腦執行（設定來源）──

# 1. 定義路徑
$src  = "$env:APPDATA\Antigravity"
$gdrive = "$env:USERPROFILE\Google Drive\My Drive\dotfiles\Antigravity"

# 2. 複製設定到 Google Drive
Copy-Item $src $gdrive -Recurse -Force

# 3. 刪除本地設定資料夾
Remove-Item $src -Recurse -Force

# 4. 建立 Junction（Windows 符號連結）
New-Item -ItemType Junction -Path $src -Target $gdrive

# 驗證
Get-Item $src  # 應顯示 LinkType: Junction
第二台（或其他台）電腦執行
# 確認 Google Drive 已同步完成後執行
$src    = "$env:APPDATA\Antigravity"
$gdrive = "$env:USERPROFILE\Google Drive\My Drive\dotfiles\Antigravity"

# 若已有設定資料夾，先備份
if (Test-Path $src) {
  Rename-Item $src "${src}_backup_$(Get-Date -f yyyyMMdd)"
}

# 建立 Junction 指向 Google Drive
New-Item -ItemType Junction -Path $src -Target $gdrive

Write-Host "✅ 同步完成！重啟 Antigravity IDE 生效。"
方法 B：手動匯出匯入 簡單快速
備份 → 傳輸 → 還原
# 備份（在舊電腦執行）
$backup = "$env:USERPROFILE\Desktop\antigravity-backup-$(Get-Date -f yyyyMMdd).zip"
Compress-Archive "$env:APPDATA\Antigravity" $backup
Write-Host "備份完成：$backup"

# 還原（在新電腦執行）
Expand-Archive "C:\path\to\antigravity-backup-xxxxxxxx.zip" "$env:APPDATA\Antigravity" -Force
方法 C：API Key 設定同步

Antigravity IDE 的 API Key 除了在 IDE 介面設定外，建議同時設定環境變數，這樣重裝 IDE 後不需要重新輸入。

PowerShell — 同時設定兩個 Key
# Anthropic（Claude 系列）
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-xxxxx", "User")

# OpenAI（GPT 系列）
[System.Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-proj-xxxxx", "User")

# Google（Gemini 系列）
[System.Environment]::SetEnvironmentVariable("GOOGLE_API_KEY", "AIzaSy-xxxxx", "User")

install.ps1 — 一鍵換機腳本
install.ps1（在新電腦執行）
# ── 黑熊老師的 AI 環境一鍵安裝腳本 ──
# 使用方式：以系統管理員執行 PowerShell，貼上此腳本

param(
  [string]$GDrivePath = "$env:USERPROFILE\Google Drive\My Drive"
)

Write-Host "🚀 開始安裝 AI 開發環境..." -ForegroundColor Cyan

# 1. 安裝 Claude Code
Write-Host "[1/4] 安裝 Claude Code CLI..."
npm install -g @anthropic-ai/claude-code

# 2. 安裝 Codex CLI
Write-Host "[2/4] 安裝 Codex CLI..."
npm install -g @openai/codex

# 3. 設定 Antigravity IDE 設定符號連結
Write-Host "[3/4] 設定 Antigravity IDE 同步..."
$antiSrc  = "$env:APPDATA\Antigravity"
$antiDst  = "$GDrivePath\dotfiles\Antigravity"
if (Test-Path $antiSrc) {
  Rename-Item $antiSrc "${antiSrc}_bak"
}
New-Item -ItemType Junction -Path $antiSrc -Target $antiDst | Out-Null

# 4. 提示設定 API Keys
Write-Host "[4/4] 請設定 API Keys：" -ForegroundColor Yellow
Write-Host '  [System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY","your-key","User")'
Write-Host '  [System.Environment]::SetEnvironmentVariable("OPENAI_API_KEY","your-key","User")'

Write-Host "✅ 安裝完成！請設定 API Keys 後重啟終端機。" -ForegroundColor Green

情境 選用工具 理由
日常教學網頁開發Antigravity IDE視覺化介面，可即時預覽，適合 HTML/CSS
複雜邏輯除錯Claude Code終端機快，直接讀寫檔案，適合多步驟任務
需要 GPT-4o 的任務Codex CLIOpenAI 模型，補足 Claude 的盲點
快速問答Antigravity Agent 面板不離開 IDE，隨問隨答
Git 部署任一工具的終端機三個工具都有終端機，指令相同
核心心法：三種工具共用同一份程式碼（Git）和同一份記憶（memory.md），只是對話介面不同。不管換哪台電腦、用哪個工具，開發進度永遠在 GitHub 上。
