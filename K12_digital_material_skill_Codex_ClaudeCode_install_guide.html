5. 建立專案規則檔

Codex：AGENTS.md

保存每次工作都必須遵守的固定規則。

Claude Code：CLAUDE.md

保存語言、技術、教材與 Git 規範。

# 數位教材專案規則

本專案為臺灣國中小數位互動教材。
- 使用繁體中文。
- 用語符合指定年級。
- 優先使用 HTML、CSS、SVG 與原生 JavaScript。
- 所有頁面支援手機、平板與電腦。
- 題目必須來自教材內容。
- 不得杜撰十二年國教課綱代碼。
- 每完成重要階段建立 Git commit。
- 輸入「收工」時，執行完整檢查並同步 GitHub。
差異：規則檔保存專案長期原則；Skill 負責完整工作流程。

6. 實際使用方式

6.1 第一次建立教材

請使用 k12-interactive-learning-builder Skill。

我要建立一份國小五年級自然科互動教材。
教材主題：水的三態變化
副主題：固態、液態與氣態、蒸發、凝結、凝固、熔化、生活應用。
請使用引導模式。
先執行 Git 初始化，再提出教材藍圖。

6.2 快速模式

使用快速模式建立國小四年級「月相變化」互動教材。
採用預設頁籤、10 題闖關題與 8 個自主學習資源。

6.3 繼續專案

繼續教材。
請讀取 project.config.json、README.md、CHANGELOG.md 與 Git 狀態，
再接續尚未完成的動畫與題庫。

6.4 教材健檢

教材健檢。
檢查內容正確性、年級適切性、RWD、SVG 動畫、測驗題、
GitHub Pages 路徑與無障礙設計。

7. GitHub 部署

7.1 登入

gh auth login
gh auth status

7.2 建立 Repository

gh repo create water-cycle-interactive-learning `
  --public `
  --source=. `
  --remote=origin `
  --push
規則說明範例
全部小寫避免大小寫混用water-cycle
使用連字號單字間不用空格plant-growth-explorer
英文主題便於部署與辨識electric-circuit-lab
不要把 API key、密碼、service role key 或其他敏感資訊提交到 GitHub。

8. 「收工」流程

收工
  1. 讀取 project.config.json。
  2. 執行內容、功能、RWD、無障礙與路徑檢查。
  3. 更新版本號、README、CHANGELOG 與測試報告。
  4. 檢查 Git 狀態並建立 commit。
  5. 先拉取遠端變更，再推送本地變更。
  6. 確認 GitHub Pages 狀態。
  7. 列出最近 Git 紀錄。
  8. 產生 Facebook 教材介紹文與結案摘要。
git status
git add .
git commit -m "release: complete interactive teaching material"
git pull --rebase origin main
git push origin main
git log --oneline --decorate -20
沒有變更時不要建立空白 commit；發生衝突時不可直接強制推送。

9. 常見問題

找不到 Skill

  • 確認資料夾名稱正確。
  • 確認 SKILL.md 位於第一層。
  • 重新啟動 Codex 或 Claude Code。
  • 確認沒有多包一層同名資料夾。

GitHub 無法建立 Repository

gh auth status
gh auth login

Git 無法提交

git config --global user.name "你的名稱"
git config --global user.email "你的Email"

GitHub Pages 顯示空白

  • 檢查 CSS、JavaScript 與圖片是否使用相對路徑。
  • 不可使用 C:\... 等本機絕對路徑。
  • 使用 Vite 時檢查 base 設定。

10. 建議工作流程

  1. 建立空白教材資料夾。
  2. 啟動 Codex 或 Claude Code。
  3. 呼叫 Skill。
  4. 提供主題、年級、科目與副主題。
  5. 確認教材藍圖。
  6. 產生網站、動畫與闖關題。
  7. 執行教材健檢。
  8. 本機預覽並修正。
  9. 部署到 GitHub Pages。
  10. 輸入「收工」。
推薦做法：全域安裝 Skill，並在每個專案保留 AGENTS.md、CLAUDE.md 與 project.config.json。
