適用對象

一、為什麼改裝 Codex CLI？

一開始使用者嘗試從 Microsoft Store 安裝 Codex 桌面版，但畫面出現「發生錯誤」。點選「查看詳細資料」後，出現：

Terminal
開啟 Windows Update
Windows Update 已停用。您可以在「設定」中執行「Windows Update 疑難排解工具」來修復 Windows Update。
代碼：0x80070422

這個錯誤通常代表 Windows Update 相關服務被停用或沒有正常啟動。Microsoft Store 安裝 App 時需要依賴 Windows Update、BITS、Delivery Optimization 等背景服務，因此當 Windows Update 被停用時，Store 版 Codex 就可能無法下載或安裝。

因此，短期可行方案是：先安裝 Codex CLI 終端機版本。Codex CLI 是 OpenAI 的命令列 coding agent，可在本機終端機中讀取、修改與執行指定資料夾中的程式碼。OpenAI 官方文件說明，ChatGPT Plus、Pro、Business、Edu、Enterprise 方案包含 Codex 使用權；也可選擇使用 API key 依用量計費。[1]

🪟

二、安裝前檢查 Windows 版本

1. 使用 winver 查看版本

按下：

Terminal
Win + R

輸入：

Terminal
winver

範例查詢結果：

Terminal
Windows 11 專業版
版本：21H2
OS 組建：22000.318

這代表電腦是 Windows 11，但版本偏早。若 Microsoft Store 安裝 App 出問題，仍建議後續修復 Windows Update，將系統更新到較新的版本。不過在本次情境中，Codex CLI 的安裝主要卡在 PowerShell 執行原則，而不是 Windows 版本本身。

🟢

三、安裝 Node.js 與 npm

Codex CLI 可透過 npm 安裝。npm 會跟著 Node.js 一起安裝。

1. 安裝 Node.js

到 Node.js 官方網站下載 LTS 或目前穩定版本，安裝時保留預設選項即可。

2. 檢查 Node.js

在 PowerShell 輸入：

Terminal
node -v

本次範例顯示：

Terminal
v24.15.0

代表 Node.js 已安裝成功。

3. 檢查 npm 時遇到錯誤

接著輸入：

Terminal
npm -v

畫面出現：

Terminal
npm : 因為這個系統上已停用指令碼執行，所以無法載入 C:\Program Files\nodejs\npm.ps1 檔案。
CategoryInfo          : SecurityError
FullyQualifiedErrorId : UnauthorizedAccess

這不是 npm 沒有安裝，而是 PowerShell 的 Execution Policy（執行原則）阻擋 .ps1 指令碼執行。Microsoft 文件說明，PowerShell 執行原則是一種安全功能，用來控制 PowerShell 載入設定檔與執行指令碼的條件，可降低惡意指令碼執行的風險。[2]

🛠️

四、解法一：用 npm.cmd 避開 PowerShell 的 ps1 限制

在 Windows 中，npm 通常同時有 .ps1 與 .cmd 版本。PowerShell 預設可能優先呼叫 npm.ps1，因此被執行原則擋住。可以改用：

Terminal
npm.cmd -v

如果能看到 npm 版本號，就可以直接安裝 Codex CLI：

Terminal
npm.cmd install -g @openai/codex

官方 GitHub README 也列出全域安裝指令：

Terminal
npm i -g @openai/codex

安裝完成後，在 PowerShell 中若直接輸入：

Terminal
codex --version

仍可能出現：

codex : 因為這個系統上已停用指令碼執行，所以無法載入 C:\Users\user\AppData\Roaming\npm\codex.ps1 檔案。

這時可改用：

Terminal
codex.cmd --version

或直接在 cmd.exe 中執行：

Terminal
codex

本次實際安裝流程中，切換到 cmd.exe 後成功進入 Codex CLI 登入畫面。

🔐

五、解法二：將 PowerShell 執行原則改成 RemoteSigned

若希望之後可以直接在 PowerShell 輸入 npm、npx、codex，而不用每次加 .cmd，可將目前使用者的 PowerShell 執行原則改成 RemoteSigned。

1. 設定指令

開啟 PowerShell，輸入：

Terminal
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

若出現確認訊息：

Do you want to change the execution policy? [Y] Yes [A] Yes to All [N] No ...

請輸入：

Terminal
Y

然後按 Enter。

2. 查詢設定是否成功

輸入：

Terminal
Get-ExecutionPolicy -List

若看到：

Terminal
Scope           ExecutionPolicy
-----           ---------------
CurrentUser     RemoteSigned

代表目前使用者範圍已設定完成。Microsoft 官方 Set-ExecutionPolicy 文件說明，此 Cmdlet 可變更 Windows 電腦上的 PowerShell 執行原則；-Scope CurrentUser 表示只針對目前使用者生效。[3]

3. 重新開啟 PowerShell 後測試

關閉 PowerShell，再重新開啟，輸入：

Terminal
npm -v
codex --version

如果兩者都能正常顯示版本資訊，代表問題已解除。之後即可直接輸入：

Terminal
codex
🚀

安裝成功後，在終端機輸入：

Terminal
codex

2. Sign in with Device Code

3. Provide your own API key

七、第一次測試 Codex CLI

建議先建立測試資料夾，不要一開始就在重要專案中操作。

Terminal
mkdir C:\codex-test
cd C:\codex-test
codex

進入 Codex 後可輸入：

請幫我建立一個簡單的 HTML 網頁，主題是「月相變化教學」，包含標題、四個月相卡片與簡單說明。

這樣可以確認 Codex 是否能在目前資料夾中建立或修改檔案。

十、安全提醒

⚠️

RemoteSigned 建議設定在 CurrentUser 範圍，不要一開始就改整台電腦的 LocalMachine。

⚠️

不要執行來源不明的 .ps1 腳本。

⚠️

Codex CLI 可以讀取、修改與執行目前專案資料夾中的程式碼，因此第一次使用建議從測試資料夾開始。

⚠️

在正式專案中使用前，建議先建立版本控制，例如 Git repository，方便還原修改。

⚠️

不要把 API Key、密碼、學校系統帳密直接貼進 Codex 對話或專案檔案中。

✅

十一、結論

本次安裝過程中，真正遇到的問題有兩層：

第一層是 Microsoft Store 版 Codex App 因 0x80070422 無法安裝，原因可能與 Windows Update 服務被停用有關。第二層是 Codex CLI 安裝後，在 PowerShell 中執行 npm、npx、codex 時被 .ps1 執行原則擋住。

最簡單的解法是：

Terminal
npm.cmd install -g @openai/codex
codex

若要讓 PowerShell 長期順利使用，則建議設定：

Terminal
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

設定完成後，Codex CLI 就可以成為 Windows 上進行 AI 程式開發、教材互動網頁設計、App 原型製作與 GitHub 部署前測試的重要工具。
