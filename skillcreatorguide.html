1什麼是 Claude Code Skill？

Skill 是一段寫給 Claude 的「作業程序說明」，存放在特定資料夾中。 當使用者輸入的訊息符合 Skill 的觸發條件時，Claude 會自動載入並依照說明執行任務， 不需要使用者每次都重複交代相同的背景與流程。

你可以把 Skill 想成：

類比說明
公司 SOP 文件把重複工作的步驟標準化，讓執行者（Claude）每次都照同一套流程做
VS Code Snippet輸入特定關鍵字，自動展開一段預設程式碼/流程
GPT Custom Instructions告訴 AI「在特定情境下，要怎麼思考、怎麼回應」
💡 Skill 與一般 prompt 的最大差別：Skill 是持久化的。你寫一次，之後每個符合條件的對話都會自動套用，不需要每次重新說明。

2Skill 的檔案結構

最精簡的 Skill 只需要一個 SKILL.md 檔案：

.claude/skills/
  your-skill-name/
    SKILL.md    # 必要：frontmatter + 操作說明

複雜的 Skill 可以加入輔助檔案：

.claude/skills/
  your-skill-name/
    SKILL.md           # 主要說明
    reference.md       # 程式碼片段、參考資料
    templates/
      example.html.template # 可套用的範本

命名規則

  • 資料夾名稱 = Skill 名稱，用小寫 + 連字號（kebab-case）
  • 命名反映「功能」而非「實作細節」
  • 範例：pdf-extractor、commit-message-writer、interactive-science-courseware

3撰寫 Frontmatter（最關鍵的一步）

SKILL.md 的開頭必須有 YAML frontmatter，格式如下： 

SKILL.md---
name: your-skill-name
description: <這裡是 Claude 判斷是否載入此 Skill 的唯一依據>
---

description 怎麼寫才有效？

description 是 Claude 比對使用者訊息時用的關鍵欄位，寫法決定 Skill 的觸發精準度。

⚠️ 常見錯誤：只寫功能、不寫觸發情境。
❌ Helps create interactive science lessons.
✅ Use this skill when the user wants to create interactive digital teaching materials for natural science using SVG animations, CSS, and JavaScript. Triggers on phrases like "做一個自然科互動教材", "SVG 動畫教學"...

好的 description 包含三個要素：

  1. 使用時機（When）
    以「Use this skill when the user wants to...」開頭，描述哪種情境下應套用此 Skill。
  2. 觸發關鍵字（Triggers）
    列出使用者可能說出的字句，中英文皆可，包含同義詞與不同說法。
  3. 功能摘要（What）
    說明這個 Skill 能做什麼，幫助 Claude 確認情境是否符合。 
SKILL.md 範例---
name: interactive-science-courseware
description: Use this skill when the user wants to create
  interactive digital teaching materials for natural science
  （自然科/理化/生物/地科）using SVG animations, CSS, and
  JavaScript. Triggers on phrases like "做一個自然科互動教材",
  "SVG 動畫教學", "互動式科學模擬",
  "make an interactive science lesson with SVG/CSS/JS".
---

4撰寫 Skill 主體內容

Frontmatter 之後是給 Claude 的操作說明，建議用以下結構組織：

標準結構

SKILL.md 主體結構# Skill 標題

一句話說明此 Skill 的目的。

## 流程

### 1. 釐清需求
說明需要向使用者確認哪些資訊，以及什麼情況下可以略過。

### 2. 執行步驟
列出逐步執行的具體動作，避免含糊描述。

### 3. 輸出規格
說明輸出物的格式、放置位置、命名規則。

## 注意事項
- 常見錯誤或邊界情況
- 限制條件

## 範例
輸入：<範例情境>
輸出：<期望結果>

撰寫原則

✅
只寫 Claude「執行時需要知道」的資訊——不要寫成完整教學文件。 把長段參考資料、程式碼片段移到 reference.md，在 SKILL.md 中用相對路徑引用。
  • 用編號清單或標題分隔步驟，讓 Claude 能循序執行
  • 說明「需要哪些輸入」和「如何取得缺少的資訊」
  • 列出 1–2 個輸入輸出範例，幫助 Claude 校準格式
  • 避免硬編碼無關的示範資料

5進階：加入輔助檔案

當 Skill 需要重複使用的程式碼、格式或大量參考資料時，應拆分為獨立檔案。

reference.md — 程式碼片段與參考資料

適合放：常用程式碼模式、API 參數說明、設計規範。在 SKILL.md 中用相對路徑引用：

## 常用互動模式
詳細程式碼片段請參考 `reference.md`。

templates/ — 可直接套用的範本

適合放：空白骨架檔案（HTML、Markdown、JSON 等），讓 Claude 能直接複製後填入內容，減少每次從零生成的時間。

reference.md 片段範例## 滑桿即時控制 SVG 屬性

```js
const slider = document.getElementById('angle');
slider.addEventListener('input', () => {
  element.setAttribute('transform', `rotate(${slider.value})`);
});
```

6放置位置與部署

範圍 路徑 說明
專案層級 <repo>/.claude/skills/<name>/SKILL.md 僅在此 repo 的 Claude Code session 中可用
使用者層級 ~/.claude/skills/<name>/SKILL.md 所有 repo 的 session 皆可使用
💡 建議先放在專案層級測試，確認行為正確後再移到使用者層級推廣給所有專案使用。

確認路徑後，將整個 .claude/ 資料夾提交到 Git，這樣團隊成員 clone repo 後就能共享同一套 Skill。

7測試與驗證

  1. 檢查 frontmatter 格式
    確認是合法 YAML，name 欄位與資料夾名稱一致。
  2. 觸發測試
    開啟新的 Claude Code session，輸入一句符合 description 觸發條件的話， 確認 Skill 被自動載入（左下角或對話中會顯示 Skill 名稱）。
  3. 完整流程測試
    走一遍 SKILL.md 中定義的完整流程，確認每個步驟可執行、引用的路徑（reference.md、templates/）正確。
  4. 邊界情況測試
    測試使用者提供資訊不完整的情況——Skill 是否會要求補充？是否有合理的預設行為？

8完整範例：interactive-science-courseware

以本次實際建置的自然科數位教材 Skill 為例，說明所有元素如何組合。

最終檔案結構

.claude/skills/interactive-science-courseware/
  SKILL.md               # 主要說明：設計流程 + SVG/CSS/JS 原則
  reference.md          # 5 種互動模式程式碼片段
  templates/
    lesson.html.template  # 可直接運行的 HTML 骨架（單擺示範）

SKILL.md frontmatter

---
name: interactive-science-courseware
description: Use this skill when the user wants to create
  interactive digital teaching materials for natural science
  （自然科/理化/生物/地科/物理/化學）using SVG animations,
  CSS, and JavaScript. Triggers on phrases like
  "做一個自然科互動教材", "SVG 動畫教學", "互動式科學模擬",
  "做一個理化動畫網頁".
---

觸發後的實際輸出

使用者說「幫我做一個關於奇妙的溶解的自然科互動教材」後， Skill 依照流程產出了一份完整的 dissolution.html，包含：

  • 三個分頁（分辨物質的方法 / 溶解的過程 / 溶解的應用）
  • SVG 動畫：物品投入水杯、粒子擴散模擬
  • 互動控制：點選物品、滑桿調整攪拌速度、播放/重置
  • 卡片式應用介紹 + 3 題互動小測驗
  • 單一自包含 HTML 檔，直接用瀏覽器開啟即可教學
🎯
這就是 Skill 的價值：你只需要說出需求，Claude 會依照預先定義的流程（釐清需求→規劃結構→SVG 設計→JS 互動→測試）完整執行， 不需要每次都重新交代「什麼是好的教材設計」「SVG 應該怎麼寫」等背景知識。
