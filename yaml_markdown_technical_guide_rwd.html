YAML 與 Markdown：語法差異、技術原理與應用場景

兩者都使用純文字撰寫，但設計目的完全不同：YAML 是給程式讀取的「資料序列化格式」，Markdown 是給人閱讀與轉換版面的「輕量標記語言」。

適合對象：教師、教材設計者、網站維護者、GitHub 使用者、AI 工具使用者、初學開發者。

一、核心差異：資料格式 vs 文件格式

🟨 YAML

重點：描述資料結構

YAML 的主要任務是讓人用容易閱讀的方式寫出資料，例如設定檔、部署參數、工作流程、環境變數、課程資料清單等。

YAML 問的是：「這份資料是什麼結構？」
🟦 Markdown

重點：撰寫可轉換的文件

Markdown 的主要任務是讓人用簡單符號寫文章，之後可轉成 HTML、PDF、簡報、文件網站或 README 頁面。

Markdown 問的是：「這段文字要怎麼呈現？」
比較項目YAMLMarkdown
本質資料序列化語言輕量標記語言
主要目的讓程式讀取資料與設定讓人撰寫可排版文件
常見副檔名.yml、.yaml.md、.markdown
輸出結果物件、陣列、字串、數字、布林值等資料結構HTML、PDF、簡報、文件頁面等可閱讀內容
最怕錯什麼縮排錯誤、冒號後空格錯誤、型別誤判標題層級混亂、清單縮排不一致、不同平台延伸語法不相容
典型使用者工程師、DevOps、網站維護者、AI/自動化工具使用者作者、教師、文件撰寫者、開發者、知識管理者

這裡開始才是 Markdown 文章正文。

三、背後技術原理：解析後產生的東西不同

YAML Parsing Pipeline

YAML 的處理流程

文字檔
.yaml
詞法分析
冒號、縮排、破折號
語法解析
建立節點樹
型別推斷
字串/數字/布林
程式物件
dict / list / object

YAML 解析器會把純文字轉成程式語言可使用的資料結構。例如 Python 會轉成 dict 與 list，JavaScript 會轉成 object 與 array。

Markdown Rendering Pipeline

Markdown 的處理流程

文字檔
.md
區塊解析
標題/段落/清單
行內解析
粗體/連結/程式碼
轉換器
HTML/PDF
可閱讀頁面
瀏覽器/文件

Markdown 解析器會把純文字標記轉成文件結構，最常見是輸出 HTML，例如 # 標題 轉成 <h1>。

資料模型差異

YAML 資料節點

Scalar：字串、數字、布林值、null。

YAML 集合節點

Sequence：有順序的清單；Mapping：鍵值對集合。

Markdown 文件節點

Heading、Paragraph、List、Link、Code Block、Table 等。

技術直覺：YAML 比較像「資料庫表單」或「設定面板的原始檔」；Markdown 比較像「文章草稿」或「網頁內容的原始稿」。

四、應用場景：什麼時候該用哪一個？

⚙️
網站或系統設定
用 YAML：例如網站名稱、選單、部署環境、Docker Compose、GitHub Actions。
📘
教學講義與技術文章
用 Markdown：例如 README、教學筆記、研習講義、技術文件。
🤖
AI 與自動化流程
YAML 常用來描述工作流程、任務參數與模型設定；Markdown 常用來輸出可讀報告。
🌐
靜態網站生成器
常同時使用：YAML Front Matter 設定 metadata，Markdown 撰寫文章正文。
🧪
課程資料庫
用 YAML 儲存課程名稱、單元、週次、材料、評量方式，方便程式讀取。
📝
知識管理與筆記
用 Markdown 寫筆記，搭配 Obsidian、HackMD、GitHub、文件網站工具。

實務選擇建議

需求建議格式原因
我要寫一篇教學文章Markdown標題、段落、清單、圖片、連結都很直覺。
我要設定 GitHub Actions 自動部署YAML工具需要讀取明確的工作流程資料結構。
我要建立課程資料給網頁讀取YAML 或 JSON資料欄位清楚，便於程式轉換與重複利用。

我要做部落格文章，同時設定日期和標籤YAML + MarkdownFront Matter 管 metadata，Markdown 管正文。

六、常見錯誤與檢查清單

YAML 常見錯誤
  • 使用 Tab 縮排，導致解析失敗。
  • 冒號後沒有空格，例如 title:課程。
  • 清單縮排層級不一致。
  • 把 yes、no、on、off 當字串時未加引號，可能產生型別問題。
  • 在不同 YAML 版本或工具中，型別解析規則不完全相同。
Markdown 常見錯誤
  • 標題層級跳太快，例如從 # 直接跳到 ###。
  • 清單與程式碼區塊縮排不一致。
  • 不同平台支援的表格、註腳、數學式語法不同。
  • 圖片路徑使用本機路徑，部署到網站後無法顯示。
  • 過度依賴 HTML 標籤，降低 Markdown 的可讀性。

七、一句話總結

YAML 是「資料的語言」，Markdown 是「文件的語言」；YAML 讓機器正確理解設定，Markdown 讓人快速寫出可閱讀的內容。
