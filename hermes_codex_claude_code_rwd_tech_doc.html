快速結論

以目前教材網站、科展文件、互動教材與專案維護需求來看。

建議學習順序：

1. Codex → 2. Claude Code → 3. Hermes Agent。 Codex 用來處理日常程式與網站修改；Claude Code 用來處理大型專案、重構與複雜除錯； Hermes Agent 則適合未來打造長期個人 AI 管家。

日常開發 Codex 最適合修網頁、改 HTML/CSS/JS、產生 README、整理 GitHub Pages。
大型重構 Claude Code 適合讀完整專案、跨多檔案修改、架構重整與長脈絡推理。
長期自動化 Hermes Agent 適合自架、記住工作流程、定期執行任務與跨平台助理。

三者一句話定位

先抓住每個工具最核心的角色。

C

Codex

程式專案助教

適合進入專案資料夾，協助寫程式、修 bug、改檔案、跑測試與整理專案。 對 GitHub Pages、HTML 簡報、互動教材與小工具製作很實用。

GitHub Pages HTML/CSS/JS README
CC

Claude Code

深度程式協作助教

適合大型專案、長上下文理解、跨檔案重構與複雜問題追蹤。 如果專案檔案很多、需求混合技術文件與程式，Claude Code 很適合處理硬仗。

大型專案 重構 長脈絡
H

Hermes Agent

自架個人 AI 管家

適合長期記憶、累積 skills、自動化例行工作與跨平台助理。 它不是單次改程式最優先，而是未來打造「黑熊老師 AI 管家」時值得研究。

長期記憶 自動化 自架

任務情境建議

依照實際工作選工具，避免每件事都用同一套。

適合用 Codex

1

修 GitHub Pages、調整版面、處理 HTML/CSS/JS 問題。

2

產生 README、安裝說明、簡單工具腳本與專案整理。

3

把教案、科展內容轉成網頁、互動教材或 HTML 簡報。

適合用 Claude Code

1

完整閱讀大型專案，找出重複元件與結構問題。

2

將教材網站改成資料驅動式架構，例如用 JSON 產生頁面。

3

處理複雜 bug、跨多檔案重構、長文件與程式混合任務。

適合用 Hermes Agent

1

長期記住科展前置作業格式、海報提示詞規則與教材網站流程。

2

定期檢查網站、整理更新紀錄、提醒尚未完成的任務。

3

透過 Telegram、Discord、Slack 等平台變成長期助理。

適合用 ChatGPT

1

發想教案、規劃課程、產生文章、潤飾語氣與整理表格。

2

先設計教材內容與資訊架構，再交給 Codex 或 Claude Code 實作。

3

產出 Word、PDF、簡報大綱、科展前置作業與教師研習文件。

建議工作流

將三者放進同一條產出流程，而不是互相取代。

1

ChatGPT：先規劃內容

用 ChatGPT 發想課程、整理架構、撰寫技術文件、設計教材內容與學生學習單。

2

Codex：把內容變成網站或工具

將內容轉成 GitHub Pages、HTML 簡報、互動教材、README、工具腳本或可部署專案。

3

Claude Code：處理大型重構與複雜除錯

當專案變大、檔案變多、需要長脈絡分析與跨檔案重構時，再使用 Claude Code。

4

Hermes Agent：未來轉成長期自動化

等流程穩定後，再讓 Hermes Agent 記住規格、累積 skills，成為長期 AI 管家。

要快速改網站？ 選 Codex。
要整理大型專案？ 選 Claude Code。
要長期自動化？ 選 Hermes Agent。
要先想教案內容？ 選 ChatGPT。

可直接使用的 Prompt 範例

依工具不同，指令也要不同。

【Codex】 請讀取這個 GitHub Pages 專案，新增一頁「科展研究目的」，維持原本網站風格，並更新 index.html 的連結。完成後請列出修改的檔案。
【Claude Code】 請完整閱讀這個 science-portal 專案，整理目前網站架構，找出重複元件，並提出可以重構成資料驅動式架構的具體方案。
【Hermes Agent】 請記住我的科展前置作業格式。未來我只要提供研究題目，就依照研究動機、研究目的、文獻探討、實驗設計、材料工具、紀錄表與參考文獻產生初稿。

最終建議

目前最實際的搭配是：ChatGPT 負責內容與架構，Codex 負責日常開發， Claude Code 負責大型專案與複雜重構，Hermes Agent 則作為未來長期自動化助理。
