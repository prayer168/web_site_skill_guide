一、會話管理

控制對話 context、切換任務、恢復舊會話

§ 01
命令 說明
/clear 清除目前對話的所有 context，切換任務時使用
/compact 將舊訊息壓縮成摘要，釋放 token 但保留脈絡
/context 以彩色方格視覺化顯示目前 context 用量
/copy [N] 複製 Claude 最後一則回應到剪貼簿（/copy N 複製第 N 則）
/resume 恢復之前儲存在磁碟上的舊會話
/branch 在目前對話點建立分支新會話（別名：/fork）
/btw 插問一個小問題，不打斷主要對話流程
⚙️

二、設定與配置

調整模型、主題、記憶體與工具權限

§ 02
命令 說明
/config 開啟設定面板
/color 設定本次會話的提示列顏色
/model 切換使用的 AI 模型（Sonnet / Opus / Haiku）
/memory 查看並管理 Claude 記憶的專案資訊
/permissions 管理工具使用權限
/autofix-permissions 自動分析使用紀錄，建立允許清單以減少確認提示
/theme 開啟主題選擇器 / 管理自訂主題
📁

三、專案與目錄

管理工作目錄與專案說明文件

§ 03
命令 說明
/add-dir 新增工作目錄讓 Claude 可讀寫
/init 初始化 CLAUDE.md（專案說明文件）

四、代理與擴充

子代理、MCP 伺服器連線、瀏覽器整合

§ 04
命令 說明
/agents 管理 agent 設定（子代理配置）

五、費用與使用統計

查看 API 花費與方案使用量

§ 05
命令 說明
/cost 顯示本次會話的 API 費用
/stats 開啟使用統計（每日用量、會話、連續使用天數）
/usage 完整使用儀表板（方案限制、費率、費用、每日統計）

小技巧

提升效率的使用技巧彙整

TIP
輸入 / 後接字母 即時篩選命令清單，快速定位所需命令
TIP
MCP 伺服器命令 連線後出現 /mcp__<伺服器>__<功能> 格式的額外命令
TIP
/help 永遠顯示最新的完整命令清單，含自訂命令
TIP
自訂命令 在 .claude/commands/ 建立 .md 檔即可新增專屬命令
KEY
/compact vs /clear /compact 保留摘要；/clear 完全清空 context
