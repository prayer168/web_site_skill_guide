網頁繪圖技術選型指南：Python、SVG、JavaScript 的角色與使用時機

本文件整理網頁製作中常見的靜態繪圖、互動動畫、資料視覺化與教材開發情境， 說明 Python、SVG、JavaScript 各自扮演的角色，並提供適合國小自然互動教材與 GitHub Pages 部署的實務建議。

一頁看懂：三者不是互相取代，而是分工合作

在網頁繪圖中，最容易混淆的是「誰負責畫圖？」、「誰負責互動？」、「誰負責資料處理？」。 實務上可以先記住這句話：

SVG：負責畫出可縮放圖形

適合圖示、流程圖、科學示意圖、電路圖、燒杯、試管、箭頭、標籤等。它是向量圖形，放大後不容易模糊。

JavaScript：負責讓圖形動起來

適合點擊、拖曳、滑桿、即時變色、動畫、模擬、小遊戲、互動測驗。它控制網頁上的 HTML、SVG 或 Canvas。

Python：負責資料與後端

適合分析實驗數據、批次產生 HTML/SVG、製作圖表、建立 API、連接資料庫、接 AI 模型。通常不直接取代瀏覽器端互動。

給教材製作者的快速結論： 若要部署在 GitHub Pages，最穩定的組合是 HTML + CSS + JavaScript + SVG。 Python 則建議放在「製作前處理」或「後端服務」階段使用。

三種工具在網頁中的主要角色

工具 核心角色 最適合的任務 在網頁中的位置 常見搭配
SVG 向量圖形描述語言 圖示、示意圖、流程圖、教材插圖、可縮放圖形 可直接寫在 HTML 裡，也可獨立成 .svg 檔 HTML、CSS、JavaScript、D3.js
JavaScript 瀏覽器端互動控制語言 點擊、拖曳、動畫、模擬、互動圖表、遊戲 瀏覽器端執行 SVG、Canvas、CSS、Chart.js、D3.js
Python 資料處理、後端、自動化與科學運算 資料分析、圖表產生、後端 API、AI、批次產生教材 通常在伺服器端或製作階段；也可透過 Pyodide 在瀏覽器執行 Flask、FastAPI、Pandas、Matplotlib、Plotly、Pyodide

SVG 的專業定位

SVG 是 Scalable Vector Graphics，屬於網頁原生支援的向量圖形技術。 它可以用 <circle>、<rect>、<path> 等元素描述圖形， 也可以被 CSS 與 JavaScript 操作。MDN 文件明確指出 SVG 可用 JavaScript 建立與操作，適合處理互動與嵌入式 SVG 內容。 

<svg viewBox="0 0 300 160" role="img" aria-label="燒杯示意圖">
  <rect x="90" y="40" width="120" height="90" rx="10" fill="none" stroke="#333" />
  <rect x="100" y="82" width="100" height="42" fill="#8b5cf6" opacity="0.75" />
  <text x="150" y="150" text-anchor="middle">紫色高麗菜汁</text>
</svg>

JavaScript 的專業定位

JavaScript 是瀏覽器端互動的核心。它可以操作 DOM，也可以控制 SVG 或 Canvas。 當學生按下按鈕、拖曳物件、調整滑桿、即時改變液體顏色時，背後通常就是 JavaScript。 

function changeLiquid(color, label) {
  document.querySelector("#liquid").setAttribute("fill", color);
  document.querySelector("#result").textContent = label;
}

Python 的專業定位

Python 在網頁教材中通常不是拿來直接畫前端互動，而是處理資料、產生內容或提供後端服務。 Flask 官方文件將 Flask 定位為輕量級 WSGI 網頁應用框架，適合快速建立 Python 後端。 若需要在瀏覽器中直接執行 Python，Pyodide 透過 WebAssembly 將 CPython 帶到瀏覽器，並支援部分科學套件。 

# Python：將實驗資料整理成網頁可讀的 JSON
import json

data = [
    {"solution": "醋", "type": "酸性", "color": "紅色"},
    {"solution": "純水", "type": "中性", "color": "紫色"},
    {"solution": "小蘇打水", "type": "鹼性", "color": "藍綠色"}
]

with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

什麼時機使用 SVG、JavaScript、Python？

1. 只需要清楚的靜態示意圖：優先 SVG

如果目標是畫出科學概念，例如電路圖、磁力線、植物構造、月相位置、實驗流程， 而且希望在手機、平板、投影幕上放大都清楚，SVG 是優先選擇。

適用範例： 紫色高麗菜汁實驗示意圖、酸性/中性/鹼性三欄整理圖、電池與燈泡連接示意、昆蟲旅館構造圖。

2. 需要點擊、拖拉、變色、動畫：SVG + JavaScript

如果圖形本身不複雜，但希望學生可以操作，例如「點醋，燒杯變紅」、「拖曳小蘇打水到試管」， 建議用 SVG 繪製畫面，再用 JavaScript 控制互動。

適用範例： 酸鹼指示劑變色、月相拖曳排序、電路開關互動、磁鐵吸引物品分類。

3. 需要大量資料、統計、圖表：Python 先處理，前端再呈現

當資料量增加，例如全班實驗紀錄、問卷、科展測量數據、聲音波形數據， 可先用 Python 的 Pandas、Matplotlib、Plotly 等工具處理與產生圖表。 Plotly Python 文件指出，互動圖表可以匯出成 HTML 檔，讓瀏覽器直接開啟。

適用範例： 雨聲筒波形分析、植物生長紀錄圖、科展問卷統計、班級實驗結果視覺化。

4. 需要後端、登入、資料庫或 AI：Python 後端

如果網站需要儲存學生作答、產生個人化回饋、連接 AI 模型、讀寫資料庫， 單純 GitHub Pages 就不夠，這時可以用 Flask、FastAPI 或其他後端框架。 不過一旦加入後端，就需要部署到支援伺服器的環境，例如 Render、Railway、Zeabur、Cloud Run 或自架伺服器。

5. 想在瀏覽器中直接跑 Python：考慮 Pyodide，但要小心載入成本

Pyodide 可以讓 Python 在瀏覽器執行，適合科學計算展示、Python 教學、離線小型運算。 但它通常比單純 JavaScript 檔案更重，對國小教材而言，除非真的需要 Python 生態系，否則不一定是第一選擇。

補充：Canvas 什麼時候比 SVG 更適合？

雖然本文件主軸是 Python、SVG、JavaScript，但網頁動畫常會遇到另一個選項：Canvas。 MDN 將 Canvas API 說明為透過 JavaScript 與 <canvas> 元素繪製圖形的方式， 可用於動畫、遊戲圖形、資料視覺化、照片處理與即時影像處理。

比較項目 SVG Canvas
圖形本質 向量圖形，每個圖形元素都是 DOM 節點 像素畫布，畫完後不保留每個物件的 DOM 節點
放大清晰度 通常較清晰，適合 RWD 與投影 需要處理解析度與縮放，否則可能模糊
互動控制 可直接選取某個圖形元素，例如某個燒杯、按鈕、箭頭 需自行計算滑鼠座標與物件碰撞
適合情境 圖示、流程圖、少量物件互動、教材示意圖 大量粒子、遊戲、連續動畫、影像處理、模擬大量物件
國小教材建議 多數互動教材優先選擇 做遊戲型或大量動畫時再考慮

選型決策表：遇到需求時怎麼選？

需求情境 優先工具 原因 是否適合 GitHub Pages
科學示意圖、流程圖、圖示 SVG 可縮放、清楚、容易嵌入網頁 非常適合
按鈕變色、拖拉、互動測驗 JavaScript + SVG SVG 負責畫面，JavaScript 負責操作 非常適合
大量粒子、遊戲、即時動畫 JavaScript + Canvas Canvas 適合高頻率重繪與大量物件 適合
統計圖、科展數據圖 Python + Plotly / Matplotlib，或 JavaScript 圖表庫 Python 適合資料處理，Plotly 可輸出互動 HTML 若輸出靜態 HTML，適合
自動生成很多單元教材 Python + HTML 模板 可批次套版，減少重複排版 產出的 HTML 適合
學生登入、儲存成績、全班資料庫 Python 後端 + 資料庫 需要伺服器處理資料與權限 GitHub Pages 不適合單獨完成
AI 回饋、文字生成、影像辨識 Python 後端或外部 API + JavaScript 前端 AI 與金鑰不宜直接放在前端 通常需要後端
瀏覽器中教 Python 或跑科學計算 Pyodide 可在瀏覽器執行 Python，但載入較重 可行，但需評估效能
