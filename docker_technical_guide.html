Docker 是什麼？
工作原理與實務應用

這份文件用「教師與教材開發者」容易理解的方式，說明 Docker 的概念、Image、Container、Dockerfile、Docker Compose，以及如何應用在網站、AI 工具與教學專案。

一句話理解 Docker

Docker 是一種容器化平台，可以把程式、執行環境、套件與設定打包成標準化的容器，讓程式在不同電腦或伺服器上穩定執行。

白話比喻：Docker 就像幫每個程式準備一間「獨立、輕量、可搬家的小房間」，裡面已經放好程式需要的工具。

01｜Docker 的核心概念

Docker 的重點不是「再開一台完整電腦」，而是把應用程式需要的環境標準化，讓它可以更容易被建立、分享與執行。

📦Image 映像檔程式環境的範本，包含作業系統基礎、執行環境、套件、程式碼與啟動設定。
🚢Container 容器由 Image 啟動後正在執行的實體環境。同一個 Image 可以開很多個 Container。
📝Dockerfile建立 Image 的說明書，告訴 Docker 要安裝什麼、複製哪些檔案、如何啟動程式。

Image 與 Container 的關係

名稱白話說法是否正在執行例子
Image環境範本、安裝包、模型屋設計圖否node:20、nginx:alpine
Container真正跑起來的程式環境是正在執行的網站、API、資料庫
Dockerfile寫出建立環境的步驟
docker build根據 Dockerfile 建立 Image
Image形成可重複使用的環境範本
docker run啟動 Container，程式開始執行

02｜Docker 的工作原理

Docker 使用作業系統層級的隔離技術，讓不同程式可以在彼此獨立的容器中執行。容器通常共享主機作業系統核心，因此比傳統虛擬機更輕量、啟動更快。

Docker 容器 vs. 虛擬機 VM

比較項目Docker 容器虛擬機 VM
啟動速度快，常見為數秒內較慢，通常需要較長啟動時間
資源消耗較低較高
作業系統共享主機核心，不包含完整 OS通常包含完整客體作業系統
適合用途開發環境、網站部署、微服務、AI 工具完整系統測試、不同 OS 模擬、企業虛擬化
重點：虛擬機像「再開一台完整電腦」；Docker 像「在同一台電腦中隔出多個獨立房間」。

Docker Compose 是什麼？

當專案不只有一個服務，例如前端、後端、資料庫、管理介面，就可以使用 Docker Compose。它用 docker-compose.yml 或 compose.yml 描述多個服務，並用一個指令啟動整套環境。

services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"

03｜Docker 常用指令速查

指令功能使用時機
docker --version查看 Docker 版本確認是否安裝成功
docker run hello-world執行測試容器第一次安裝後測試
docker ps查看正在執行的容器確認服務是否正在跑
docker ps -a查看所有容器包含已停止的容器
docker images查看本機映像檔確認下載或建立過哪些 Image
docker stop 容器ID停止容器暫停服務
docker start 容器ID重新啟動容器恢復已停止服務
docker rm 容器ID刪除容器清理不用的容器
docker rmi 映像檔ID刪除映像檔釋放硬碟空間
docker compose up -d背景啟動多服務專案啟動網站＋資料庫等整套環境
docker compose down關閉並移除 Compose 容器停止整套專案

04｜實作範例：把教材網站包成 Docker

假設你有一個靜態教材網站，資料夾如下：

science-portal/
├─ index.html
├─ style.css
├─ script.js
└─ Dockerfile

步驟 1：建立 Dockerfile

FROM nginx:alpine
COPY . /usr/share/nginx/html

步驟 2：建立 Image

docker build -t science-portal .

步驟 3：啟動網站

docker run -d -p 8080:80 science-portal

步驟 4：打開瀏覽器

http://localhost:8080
成果：你的教材網站會在容器中的 Nginx 伺服器上執行。未來換一台電腦，只要安裝 Docker，重複相同指令即可啟動。

05｜Docker 可以如何應用在教學與專案？

🌐教材網站部署將 HTML、CSS、JavaScript 教材包成容器，用 Nginx 快速啟動。
🤖AI 工具環境常見 AI 工具如 Open WebUI、n8n、Flowise、資料庫服務，常用 Docker Compose 管理。
🧪學生實驗環境讓不同電腦使用相同 Node.js、Python 或資料庫版本，降低安裝失敗問題。
🗂️專案交付將 Dockerfile、Compose、README 一起交付，讓他人能快速重現環境。

適合使用 Docker 的情境

情境建議原因
Node.js / Python 專案很適合可固定版本與套件環境
網站＋資料庫很適合可用 Compose 一次啟動多服務
AI 工具與 RAG 系統很適合常需要資料庫、模型服務、管理介面
單純打開 HTML 檔不一定需要直接用瀏覽器即可
Word、PPT、PDF 文件處理通常不需要除非要做自動化轉檔服務

06｜使用 Docker 的注意事項

  • 不要把密碼寫死在 Dockerfile：資料庫密碼、API Key 應用環境變數或 secret 管理。
  • 盡量使用官方 Image：例如 node、nginx、python 等官方來源。
  • 定期更新 Image：太舊的基礎映像檔可能包含已知漏洞。
  • 容器不是萬能安全保證：仍要注意權限、網路、檔案掛載與外部連線。
  • 注意硬碟空間：長期使用後，可定期清理不用的容器與映像檔。
# 清理未使用的容器、網路、映像檔與快取
 docker system prune
提醒：執行清理指令前，請先確認沒有要保留的容器、Image 或 volume，避免誤刪專案資料。

07｜初學者學習路線

1安裝 Docker Desktop
確認 Windows、macOS 或 Linux 環境可以執行 Docker。
2跑 hello-world
用最簡單指令確認 Docker Engine 可正常運作。
3跑 Nginx 網站
理解 port 對應與容器服務啟動。
4學 Dockerfile
開始把自己的專案打包成 Image。
5學 Compose
一次管理網站、API、資料庫等多服務。
6套用到教材網站
把自己的 HTML / JS 教材包成可移植環境。
7嘗試 AI 工具
用 Docker Compose 啟動 Open WebUI、n8n、資料庫等。
8整理 README
寫下啟動、關閉、更新與備份方式，方便交接。
