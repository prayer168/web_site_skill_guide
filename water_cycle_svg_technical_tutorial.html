一、教學與技術目標

科學概念
呈現蒸發、凝結、降雨、地表逕流、地下滲透與回到海洋的因果關係。
動畫表現
使用非線性速度、不同時間軸與多層景深，避免物件同步、固定速度移動。
教材應用
可直接嵌入 HTML、投影片網頁、H5P、學校網站或互動式教材。

15 秒動畫腳本

時間畫面動作科學說明
0–3 秒太陽發光，海面波浪，水蒸氣上升。太陽提供熱能，使海水或地表水蒸發成水蒸氣。
3–6 秒水蒸氣上升並逐漸聚集成雲。水蒸氣上升遇冷，凝結成小水滴形成雲。
6–9 秒雲層受風移動到山區。風會帶動雲移動，讓水分從海洋輸送到陸地。
9–11 秒雨滴自然落下。雲中水滴變大變重後，以降雨形式回到地表。
11–13 秒水沿山坡、河流流回海洋。地表逕流會沿地勢較低處流動，匯入河流。
13–15 秒地下水緩慢滲透並回到海洋。部分雨水滲入土壤，成為地下水並緩慢流動。

二、提示詞設計流程

產生高品質 SVG 動畫時，建議不要只要求「做一個水循環動畫」，而是把需求拆成四層：科學內容、畫面構圖、動畫規則與技術規格。

可直接使用的提示詞範本

請建立一個符合國小自然科學教學的 SVG 動畫，主題為「大自然中的水循環」。

畫面包含：太陽、海洋、河流、山脈、雲、樹木與植被。
需呈現：蒸發、凝結、降雨、地表逕流、地下滲透、回到海洋。

動畫要求：
- 太陽持續發光
- 海面有波浪效果
- 水蒸氣由海面向上漂浮
- 雲逐漸形成並移動至山區
- 雨滴自然落下
- 河流流向海洋
- 地下水以半透明藍色箭頭表示
- 全流程循環播放，總長 15 秒

技術要求：
- 使用 SVG + CSS Keyframes
- 每個元素獨立群組 <g> 並加上 id
- 使用 cubic-bezier，禁止單純線性動畫
- 雲層、降雨、河流各自獨立時間軸
- 多層景深，避免所有物件同步運動
- 不使用 Canvas
- 輸出完整 HTML 檔案，並附中文註解

三、SVG 結構設計

建議將 SVG 拆成獨立群組，這樣每個自然現象都能被單獨控制、重複播放或改成互動式按鈕觸發。

<svg id="water-cycle-scene" viewBox="0 0 1200 700">
  <g id="background-layer">天空、遠山、地面</g>
  <g id="sun-layer">太陽與光暈動畫</g>
  <g id="ocean-layer">海洋與波浪動畫</g>
  <g id="evaporation-layer">水蒸氣上升動畫</g>
  <g id="cloud-layer">雲形成與移動動畫</g>
  <g id="rain-layer">雨滴落下動畫</g>
  <g id="runoff-layer">地表逕流與河流動畫</g>
  <g id="groundwater-layer">地下滲透與地下水箭頭</g>
  <g id="label-layer">中文標籤與教學提示</g>
</svg>
技術重點：畫面物件要先分層，再設定不同 keyframes、duration、delay 與 easing。這比單一大型 path 更容易做出準確、自然的動畫。

七、實作指南：從 AI 生成到教材使用

步驟 1：先寫動畫腳本
先決定每個時間點呈現哪個水循環概念，不要一開始就寫程式。
步驟 2：建立 SVG 分層
太陽、雲、雨、河流、地下水都要用不同 <g> 群組。
步驟 3：加入 CSS Keyframes
用 transform、opacity、stroke-dashoffset、filter 等屬性控制動畫。
步驟 4：調整自然感
使用 cubic-bezier、不同 delay、不同 duration，避免所有物件同時移動。
步驟 5：加入互動
用按鈕切換 class，讓學生點擊後播放單一階段動畫。
步驟 6：加入科學說明
每個階段附上「現象、原因、關鍵詞」，提升教學可用性。

常用 CSS 技巧

/* 非線性上升：模擬水蒸氣不是固定速度往上飄 */
.vapor {
  animation: vaporRise 4s cubic-bezier(.2,.7,.3,1) infinite;
}

/* 線條流動：可用於河流、地下水、箭頭 */
.flow-line {
  stroke-dasharray: 12 10;
  animation: flow 3s cubic-bezier(.45,0,.25,1) infinite;
}

/* 互動式階段：點擊按鈕後只啟動對應動畫 */
.scene[data-stage="rain"] #rain-layer {
  animation-play-state: running;
}

教學應用建議

  • 播放基礎版後，請學生說出水循環的順序。
  • 播放進階版後，請學生觀察「雲為什麼會移動到山區」。
  • 使用互動版進行分組任務：每組負責解釋一個階段。
  • 延伸 PBL：讓學生設計「如果森林減少，水循環會如何改變？」
